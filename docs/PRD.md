# InvoiceHub MVP PRD

## 🎯 핵심 정보

**목적**: Notion "Invoices" 데이터베이스를 백엔드로 활용하여 견적서·청구서를 조회·검색·필터링하는 1인 사업자/프리랜서용 관리 웹앱
**사용자**: 청구서를 Notion에서 관리하는 1인 사업자 / 프리랜서 (본인 단독 사용)

---

## 🚶 사용자 여정

```
1. 홈 페이지 (청구서 목록 + 통계 카드)
   ↓ 상태 탭 클릭 (대기/승인/거절) 또는 검색어 입력

2. 필터링된 청구서 목록
   ↓ 청구서 카드 클릭

3. 청구서 상세 페이지
   (클라이언트·금액·기간 메타 + 항목 테이블)
   ↓ "← 목록으로 돌아가기" 클릭

4. 홈 페이지 복귀
```

```
[작성자 흐름]
Notion에서 청구서 작성/편집
   ↓ 상태(status) 필드 변경

InvoiceHub에 자동 반영 (ISR 갱신)
   ↓

홈 페이지에서 최신 청구서 목록 확인
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 청구서 목록 조회 | Notion Invoices DB에서 전체 청구서를 카드 형태로 표시 | 앱의 핵심 진입점 | 홈 페이지 |
| **F002** | 청구서 상세 조회 | 청구서 메타 정보 + Items 관계 DB 항목 테이블 렌더링 | 청구서 내역 확인의 핵심 기능 | 청구서 상세 페이지 |
| **F003** | 상태 필터링 | 대기/승인/거절 탭으로 해당 상태 청구서만 목록 표시 | 업무 상태별 빠른 조회 | 홈 페이지 |
| **F004** | 검색 기능 | 클라이언트명·견적서 번호 키워드 검색 | 특정 청구서 빠른 탐색 | 홈 페이지 |
| **F005** | 요약 통계 대시보드 | 전체 건수 / 승인 금액 합계 / 대기 건수 카드 | 청구서 현황 한눈에 파악 | 홈 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F010** | Notion API 연동 | `@notionhq/client v5.x dataSources.query`로 Invoices/Items DB 조회 | 모든 기능의 데이터 소스 | 홈 페이지, 청구서 상세 페이지 |
| **F011** | 반응형 레이아웃 | 모바일/태블릿/데스크톱 뷰포트에 맞춘 레이아웃 | 다양한 환경 지원 | 전체 페이지 |

### 3. MVP 이후 기능 (제외)

- 청구서 직접 생성/편집 (Notion에서 직접 관리)
- PDF 출력/다운로드
- 이메일 발송 기능
- 결제 연동
- 다크 모드 토글
- 다중 사용자 권한 관리

## 📱 메뉴 구조

```
📱 InvoiceHub 내비게이션

🏠 헤더 (전체 공통)
└── InvoiceHub 로고 (홈 링크)

📄 홈 페이지 (루트 /)
├── 통계 카드 3종: F005
│   ├── 전체 청구서 건수
│   ├── 승인 금액 합계 (₩)
│   └── 대기 중 건수
├── 상태 필터 탭: F003 (전체 / 대기 / 승인 / 거절)
├── 검색 바: F004 (클라이언트명·견적서 번호)
└── 청구서 카드 그리드: F001

📄 청구서 상세 (/invoice/[id])
├── 메타 정보 헤더: F002 (클라이언트명, 발행일, 유효기간, 상태)
├── 항목 테이블: F002, F010 (항목명·단가·수량·금액 + 총합계 행)
└── "← 목록으로 돌아가기" 버튼
```

## 📄 페이지별 상세 기능

### 홈 페이지

> **구현 기능:** `F001`, `F003`, `F004`, `F005`, `F010`, `F011` | **URL:** `/`

| 항목 | 내용 |
|------|------|
| **역할** | 앱 진입점 — 청구서 현황 통계 + 목록 표시 및 탐색 |
| **진입 경로** | 루트 URL 직접 접근, 헤더 로고 클릭 |
| **사용자 행동** | 통계 확인, 상태 탭 클릭, 검색어 입력, 청구서 카드 클릭 |
| **주요 기능** | • 통계 카드 3종 (전체/승인 금액/대기) (F005)<br>• Notion Invoices DB 전체 청구서 카드 그리드 (F001, F010)<br>• 발행일 내림차순 정렬<br>• 상태 필터 탭 — URL `?status=` (F003)<br>• 검색 바 — URL `?q=` (F004)<br>• 결과 없을 때 EmptyState UI<br>• SSG + ISR(`revalidate: 60`) |
| **다음 이동** | 청구서 카드 클릭 → 청구서 상세 페이지 |

---

### 청구서 상세 페이지

> **구현 기능:** `F002`, `F010`, `F011` | **URL:** `/invoice/[id]`

| 항목 | 내용 |
|------|------|
| **역할** | 개별 청구서 메타 정보 + 항목 테이블 표시 |
| **진입 경로** | 홈 페이지 청구서 카드 클릭 |
| **사용자 행동** | 청구서 내역 확인, 뒤로가기 클릭 |
| **주요 기능** | • Notion 청구서 페이지 + Items 관계 DB 병렬 조회 (F010, F002)<br>• 메타 정보 헤더 (클라이언트명, 발행일, 유효기간, 상태 배지)<br>• 항목 테이블 (항목명·단가·수량·금액) + 총합계 행<br>• 존재하지 않는 ID → `notFound()` 404 처리<br>• `generateMetadata()` — 견적서 번호 + 클라이언트명 동적 title<br>• SSG + ISR(`revalidate: 3600`) |
| **다음 이동** | "← 목록으로 돌아가기" → 홈 페이지 |

## 🗄️ 데이터 모델

### Invoice (청구서)

| 필드 | 설명 | 타입 |
|------|------|------|
| id | Notion 페이지 고유 식별자 | UUID (Notion) |
| invoiceNumber | 견적서 번호 | String (title) |
| clientName | 클라이언트명 | String (rich_text) |
| totalAmount | 총금액 | Number |
| status | 상태 | `"대기"` \| `"승인"` \| `"거절"` |
| issueDate | 발행일 | String (ISO 8601) |
| validUntil | 유효기간 | String (ISO 8601) |
| items | 관련 항목 목록 | InvoiceItem[] |

### InvoiceItem (항목)

| 필드 | 설명 | 타입 |
|------|------|------|
| id | Notion 항목 페이지 고유 식별자 | UUID (Notion) |
| name | 항목명 (제품/서비스명) | String (title) |
| unitPrice | 단가 | Number |
| quantity | 수량 | Number |
| totalPrice | 금액 (단가 × 수량) | Number (formula) |

## 🛠️ 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16.2.6** (App Router + ISR) — 정적 생성 + 점진적 재검증으로 Notion 변경 자동 반영
- **TypeScript 5** — 타입 안전성 보장
- **React 19** — UI 라이브러리

### 스타일링 & UI

- **TailwindCSS v4** (`@import "tailwindcss"` 방식, 설정 파일 없음) — 유틸리티 CSS
- **shadcn/ui** (new-york 스타일) — 고품질 React 컴포넌트
- **Lucide React** — 아이콘

### CMS & 데이터

- **@notionhq/client v5.21.0** — Notion 공식 API SDK
  - v5.x 파괴적 변경: `databases.query` 제거 → `dataSources.query({ data_source_id })` 사용
- **환경 변수**: `NOTION_API_KEY`, `NOTION_DATABASE_ID`, `NOTION_INVOICES_SOURCE_ID`, `NOTION_ITEMS_SOURCE_ID`
- **Fallback**: 환경 변수 미설정 시 `src/mocks/invoices.ts` 더미 데이터 반환

### 배포 & 호스팅

- **Vercel** — Next.js 16 최적화 배포 플랫폼 (ISR 지원)
