# 개인 개발 블로그 MVP PRD

## 🎯 핵심 정보

**목적**: Notion을 CMS로 활용하여 글 작성 즉시 블로그에 자동 반영되는 개인 기술 블로그 구축
**사용자**: 기술 블로그를 운영하는 1인 개발자 (작성자 본인 + 글을 읽는 독자)

---

## 🚶 사용자 여정

```
1. 홈 페이지 (글 목록)
   ↓ 카테고리 클릭 또는 검색어 입력

2. 필터링된 글 목록
   ↓ 글 카드 클릭

3. 글 상세 페이지
   ↓ 읽기 완료 후

4. 완료 → [홈으로 돌아가기] 또는 [다른 글 읽기]
```

```
[작성자 흐름]
Notion에서 글 작성/편집
   ↓ Status를 "발행됨"으로 변경

블로그에 자동 반영 (ISR/SSG 갱신)
   ↓

독자가 홈 페이지에서 최신 글 확인
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 글 목록 조회 | Notion DB에서 발행된 글 목록을 가져와 카드 형태로 표시 | 블로그의 핵심 진입점 | 홈 페이지, 카테고리 페이지 |
| **F002** | 글 상세 조회 | Notion 페이지 본문을 렌더링하여 개별 글 표시 | 블로그의 핵심 콘텐츠 소비 기능 | 글 상세 페이지 |
| **F003** | 카테고리 필터링 | 선택한 카테고리에 해당하는 글만 목록 표시 | 콘텐츠 탐색 편의성 | 홈 페이지, 카테고리 페이지 |
| **F004** | 검색 기능 | 글 제목 및 태그 기반 키워드 검색 | 원하는 글 빠른 탐색 | 홈 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F010** | Notion API 연동 | @notionhq/client를 통해 데이터베이스 및 페이지 콘텐츠 조회 | 모든 기능의 데이터 소스 | 홈 페이지, 글 상세 페이지, 카테고리 페이지 |
| **F011** | 반응형 레이아웃 | 모바일/태블릿/데스크톱 뷰포트에 맞춘 레이아웃 | 다양한 환경의 독자 지원 | 홈 페이지, 글 상세 페이지, 카테고리 페이지 |

### 3. MVP 이후 기능 (제외)

- 댓글 시스템 (Giscus 등)
- 다크 모드 토글
- RSS 피드
- 조회수 카운터
- 목차(TOC) 자동 생성
- 이메일 뉴스레터 구독
- 소셜 공유 버튼

## 📄 페이지별 상세 기능

### 홈 페이지

> **구현 기능:** `F001`, `F003`, `F004`, `F010`, `F011` | **메뉴 위치:** 헤더 > 홈

| 항목 | 내용 |
|------|------|
| **역할** | 블로그 진입점 — 최신 발행 글 목록 표시 및 탐색 |
| **진입 경로** | 루트 URL 직접 접근, 헤더 홈 링크 클릭 |
| **사용자 행동** | 글 목록 스크롤, 카테고리 필터 클릭, 검색어 입력, 글 카드 클릭 |
| **주요 기능** | • Notion DB에서 Status="발행됨"인 글만 조회 (F010)<br>• 글 카드 목록 표시 (제목, 카테고리, 태그, 발행일) (F001)<br>• 카테고리 필터 버튼 (F003)<br>• 제목/태그 키워드 검색 입력창 (F004)<br>• **글 카드 클릭** → 글 상세 페이지 이동 |
| **다음 이동** | 글 카드 클릭 → 글 상세 페이지, 카테고리 클릭 → 카테고리 페이지 |

---

### 글 상세 페이지

> **구현 기능:** `F002`, `F010`, `F011` | **메뉴 위치:** 홈 페이지에서 글 카드 클릭

| 항목 | 내용 |
|------|------|
| **역할** | 개별 블로그 글 본문 렌더링 및 표시 |
| **진입 경로** | 홈 페이지 또는 카테고리 페이지에서 글 카드 클릭 |
| **사용자 행동** | 글 본문 읽기, 뒤로가기 클릭, 태그 클릭으로 관련 글 탐색 |
| **주요 기능** | • Notion 페이지 블록 콘텐츠 조회 및 HTML 렌더링 (F010, F002)<br>• 글 메타 정보 표시 (제목, 카테고리, 태그, 발행일) (F002)<br>• Notion 블록 타입 지원: 텍스트, 제목, 코드 블록, 이미지, 리스트 (F002)<br>• **← 목록으로** 버튼 |
| **다음 이동** | 뒤로가기 → 홈 페이지, 태그 클릭 → 홈 페이지(태그 필터 적용) |

---

### 카테고리 페이지

> **구현 기능:** `F001`, `F003`, `F010`, `F011` | **메뉴 위치:** 헤더 > 카테고리 또는 홈 카테고리 필터 클릭

| 항목 | 내용 |
|------|------|
| **역할** | 선택한 카테고리에 속한 글 목록만 필터링하여 표시 |
| **진입 경로** | 헤더 카테고리 메뉴 클릭, 홈 페이지 카테고리 필터 버튼 클릭, 글 상세의 카테고리 배지 클릭 |
| **사용자 행동** | 카테고리별 글 목록 탐색, 다른 카테고리로 전환, 글 카드 클릭 |
| **주요 기능** | • 선택 카테고리 명칭 헤더 표시 (F003)<br>• Notion DB에서 해당 카테고리 글만 필터 조회 (F010, F003)<br>• 필터링된 글 카드 목록 표시 (F001)<br>• 카테고리 전환 탭/버튼 (F003)<br>• **글 카드 클릭** → 글 상세 페이지 이동 |
| **다음 이동** | 글 카드 클릭 → 글 상세 페이지, 다른 카테고리 클릭 → 해당 카테고리 페이지 |

## 🗄️ 데이터 모델

### BlogPost (블로그 글)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | Notion 페이지 고유 식별자 | UUID (Notion) |
| title | 글 제목 | String |
| category | 카테고리 | String (select) |
| tags | 태그 목록 | String[] (multi_select) |
| publishedAt | 발행일 | Date |
| status | 상태 (초안/발행됨) | String (select) |

### BlockContent (본문 블록)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 블록 고유 식별자 | UUID (Notion) |
| type | 블록 타입 (paragraph, heading, code 등) | String |
| content | 블록 텍스트/데이터 | String |
| postId | 소속 글 | → BlogPost.id |

## 🛠️ 기술 스택 (최신 버전)

### 🎨 프론트엔드 프레임워크

- **Next.js 15** (App Router + ISR) — 정적 생성 + 점진적 재검증으로 Notion 변경 자동 반영
- **TypeScript 5.6+** — 타입 안전성 보장
- **React 19** — UI 라이브러리

### 🎨 스타일링 & UI

- **TailwindCSS v4** (설정파일 없는 새로운 엔진) — 유틸리티 CSS 프레임워크
- **shadcn/ui** — 고품질 React 컴포넌트 라이브러리
- **Lucide React** — 아이콘 라이브러리

### 📝 CMS & 데이터

- **@notionhq/client** — Notion 공식 API SDK (데이터베이스 조회, 페이지 블록 조회)
- **notion-to-md** — Notion 블록을 Markdown으로 변환

### 🚀 배포 & 호스팅

- **Vercel** — Next.js 15 최적화 배포 플랫폼 (ISR 지원)

### 📦 패키지 관리

- **npm** — 의존성 관리
